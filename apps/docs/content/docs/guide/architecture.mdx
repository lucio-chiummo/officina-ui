```
React 19 (SPA, Vite)
  └─ TanStack Router (file-based, Zod search params, beforeLoad guards)
      └─ Feature modules (auth, users, billing, settings, …)
          ├─ React Hook Form + Zod
          ├─ TanStack Query
          ├─ Zustand client slices
          └─ @officina/ui primitives + our wrappers
  └─ API adapter layer (src/lib/api/*)
      ├─ ky instance (single fetch entry point)
      ├─ Zod safeParse at the network boundary
      └─ AppError normalization
  └─ MSW handlers (dev / test) — drop-in for the real API
```

## Key rules

- Feature modules are vertical slices. A feature owns its routes, schemas,
  hooks, components, tests, and stories.
- The API adapter is the **only** place `fetch` / `ky` lives. UI code never
  calls the network directly.
- Server state → TanStack Query. Client state (theme, sidebar, palette) → Zustand.
  URL state (table filters, pagination, search) → TanStack Router search params.
- Zod is the source of truth for entity shapes. Types are derived: `type X = z.infer<typeof XSchema>`.
