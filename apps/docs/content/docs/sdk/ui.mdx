`@officina/ui` exposes the first stable SDK primitive set. It is the package
consumers install when they want Officina styling without copying the full
template source.

## Install

```bash
pnpm add @officina/ui react react-dom @tanstack/react-table
```

Import CSS once:

```tsx
import '@officina/ui/tokens.css';
import '@officina/ui/styles.css';
```

`tokens.css` must load before `styles.css` so component styles can read the
semantic variables.

## First Component

```tsx
import { Badge, Button, Card, CardContent, CardHeader, CardTitle } from '@officina/ui';

export function WorkspaceCard() {
  return (
    <Card>
      <CardHeader>
        <CardTitle>Workspace</CardTitle>
      </CardHeader>
      <CardContent className="grid gap-3">
        <Badge tone="success">Active</Badge>
        <Button>Invite member</Button>
      </CardContent>
    </Card>
  );
}
```

## Form Recipe

```tsx
import { Button, Field, Form, Input, Select, Switch } from '@officina/ui';

export function SettingsForm() {
  return (
    <Form onSubmit={(event) => event.preventDefault()}>
      <Field>
        Workspace name
        <Input name="workspace" defaultValue="Acme Operations" />
      </Field>
      <Field>
        Plan
        <Select name="plan" defaultValue="growth">
          <option value="starter">Starter</option>
          <option value="growth">Growth</option>
          <option value="enterprise">Enterprise</option>
        </Select>
      </Field>
      <Switch label="Weekly reports" defaultChecked />
      <Button type="submit">Save changes</Button>
    </Form>
  );
}
```

## DataTable Recipe

```tsx
import { Badge, DataTable } from '@officina/ui';
import type { ColumnDef } from '@tanstack/react-table';

type User = {
  email: string;
  name: string;
  status: 'Active' | 'Invited';
};

const columns: Array<ColumnDef<User>> = [
  { accessorKey: 'name', header: 'Name' },
  { accessorKey: 'email', header: 'Email' },
  {
    accessorKey: 'status',
    header: 'Status',
    cell: ({ row }) => (
      <Badge tone={row.original.status === 'Active' ? 'success' : 'warning'}>
        {row.original.status}
      </Badge>
    ),
  },
];

export function UsersTable({ users }: { users: User[] }) {
  return <DataTable columns={columns} data={users} />;
}
```

## Ownership Rule

The UI package owns visual primitives and token CSS. Apps and feature packages
own route state, server state, validation schemas, mutations, and business copy.
If a primitive needs TanStack Router, Zustand, app-specific i18n, or generated
route trees, keep it in the template until it can be adapter-driven.

## Template vs SDK

Use `@officina/ui` in consumer apps. Use `@/components/primitives` only while
customizing the full source template. The [Primitive examples](/docs/components)
page marks which primitives are SDK-stable and which are source-only.
